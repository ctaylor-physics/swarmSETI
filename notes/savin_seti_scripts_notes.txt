*Notes on Savin's Codes*
___________________________________________________________
An example execution follows this process:  
1. collect_tarball_info.py builds a table that contains details about the observing configuration for each FRB target and maps metadata files to the location of raw data files for code execution in the next step. 
2. upchannelize_drx_high-freq.py calls hdfWaterfall.py (from commissioning) to upchannelize the data, then calls read_lwa-hdf5.py 
3. drift_search_ind.py searches through the single tuning h5 files produced by upchannelize_drx_high-freq.py for technosignatures using turboseti. 

Not all of the codes here are useful, but a good starting point... See below:
getinfo_from_tarball.py:
	- simply reports the observation configuration and information about a given tarball. This is borrowed directly from an older version of lsl with minor changes. 
	
collect_tarball_info.py:
	- modified version of getinfo_from_tarball.py that instead saves it to a metadata csv file. The file 'metadata.csv' is used as a tool to drive iterative processing of the above codes. 
	- metadata.csv, metadata_lwa1.csv, and metadata_lwa-sv.csv all appear to be made by this code. 

upchannelize_drx_high-[freq,time].py:
	- This is a pair of codes, where one upchannelizes the raw DRX data into either a high frequency or time resolution hdf5 file. 
	- After writing the channelize hdf5, the file is converted from LWA hdf5 format to the blimpy hdf5 file format. Basic implementation using os, hdfWaterfall.py, and read_lwa-hdf5.py
	- Freq: 3.0s , 2.36 Hz
	- Time: 0.2ms, 4.83 kHz
	- For this project we are going to focus on the high frequency resolution data 
	
read_lwa-hdf5.py:
	- Takes the hdf5 file produced by hdfWaterfall.py, splits the data on a per-tuning basis, and writes it out to a new hdf5 with a different format. 
	- This entails converting them into 'FILTERBANK' files with additional data attributes.
	- Really does not do much with the data array from the LWA programs, but changes the names of array axes and things. 
	
drift_search_ind.py:
	- Main script to process the products of read_lwa-hdf5.py using the blimpy pipeline.
	- This is an incomplete implementation so I expect that it is of little use for us. 
	- We will want to use bliss dedrift for this project which is the next step along the SETI path, so this is largely just a conceptual notebook. 
	
eyeDiagram.py:
	- This code is directly from lsl to view snippets of data. I think this was just used for data inspection at the time, but I am not sure. 

generate_h5.py: 
	- Build an artificial dataset for SETI testing that is somewhat based on the sensitivity and frequency resolution of this project. 
	- I have used this as a guide to help build testing data for this project.
